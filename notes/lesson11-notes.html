Summary for Lesson 23

The if statement is the main conditional statement in JavaScript. In this conditional statement we can also add and else keyword to add additional commands if the IF statement is not met. A conditional expression usually includes two balues to be compared and a conditional operator to compare the values. Some conditional operators are:

  • == — Is equal to
  • != — Is not equal to
  • < — Is less than
  • > — Is greater than
  • >= — Is greater than or equal to
  • <= — Is less than or equal to

We can also use logical operators such as ||(OR), &&(AND), and !(NOT)

Examples:

  • if ((phone == "") || (email == "")) alert("Something Is Missing!");
  • if ((phone == "") && (email == "")) alert("Both Values Are Missing!");
  • if (phone != "") alert("phone is OK"); or if (!phone == "") alert("phone is OK");

Shorthand Conditional Expressions:

variable= (condition) ? (value if true) : (value if false); value = (a == 1) ? 1 : 0; //if the value is 1 make it 1, else 0 is the same as if (a == 1) { value = 1; } else { value = 0; }

There are multiple ways to make loops/checking points. One is the switch statement, and as the name suggests, includes an initial statement and swtiches if that statement does not match. For loops uses a variable to keep track of how many times the loop has executed and stops when it reaches the desired number. While uses can be like for loops but they continue the loop as long as the condition is true. A do...while loop will be tested at the end of the loop rather than the beginning, meaning that the commands inside the loop will always execute at least once.

Loops are an excellent way to write up repetative tasks. It is especially usuful when working with arrays.

Event handlers are among the most powerful features of JavaScript. To utilize it we do not need to use the script tag. Instead we use the event name and code to invoke the event handler as an attribute of an individual HTML tag. For example:

This is a link. We can also use it to call a function: Move the mouse over this link.

Event handlers are stored as properties of the document object or another object that can receive an event. There are many events you can listen for. The click event can be used to invoke all sorts of actions. These are some of the most commonly used ones:

  • focus — When the element is focused on with a mouse or keyboard (tab).
  • blur — When the element loses focus.
  • click — When the element is clicked.
  • dblclick — When the element is clicked twice.
  • keydown — When a key is pressed down.
  • keyup — When a key is released.
  • keypress — When a key is pressed, relaying the key value.
  • mouseover — When the mouse pointer is over the element.
  • mouseout — When the mouse pointer is moved off an element after hovering.
  • mousemove — When the mouse moves.
  • mousedown — When the mouse button is pressed over an element.
  • mouseup — When the mouse button is released over an element.
  • load — When the element has fully loaded.

Common properties of the event object:

  • event.modifiers — A flag that indicates which modifier keys (Shift, Ctrl, Alt, etc.) were held down during the event. This value is an integer that combines binary values representing the different keys.
  • event.pageX — The x coordinate of the event within the web page.
  • event.pageY — The y coordinate of the event within the web page.
  • event.which — The key code for keyboard events (in Unicode) or the button that was pressed for mouse events.
  • event.button — The mouse button that was pressed. The left button’s value is 0, and the right button’s value is 2. If the mouse has a middle button, the value is 1.
  • event.target — The object where the event occurred.

We can keep track of keybaord actions with the event handler keypress.

Properties of the window Object:

  • window.closed — Indicates whether the window has been closed. Useful when working with multiple windows, as the current window containing the script cannot be closed without ending the script.
  • window.name — Specifies the name for a window opened by a script.
  • window.opener — In a new window opened by a script, provides a reference to the window containing the script that opened it.
  • window.outerHeight and window.outerWidth — Specify the height and width, respectively, of the outside of a browser window.
  • window.screen — Provides a child object that stores information about the screen the window is in, including resolution, color depth, and more.
  • window.self — Acts as a synonym for the current window object.
