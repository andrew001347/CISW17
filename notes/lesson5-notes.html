Summary for Lesson 9

Margins and padding are for adding space around elements. Align and float are for placing your elements in relation to others. Margins add space outside of your element's boarders. Padding will add space inside the boarder. To set margin's value, you can target them specifically by doing margin-top, or margin-left. You can also put them together by placing the values from top, right, bottom, and left, like a compass. Same principles apply with padding.

Elements will float horizontally, not vertically. This will sometimes cause problems when you are placing other elements. A work around this would be to use the clear property. Clear will ensure that no other floating elements will be there. For example, clear: left will make sure no other floating elements are allowed to the left.

Summary for Lesson 10

The CSS Box Model

Every element in HTML has a box around them. How this box is enclosed is (going outside to inside) margin, border, padding, and then the content itself.

  • The margin is the area outside the element. It never has color; it is always transparent.
  • The border extends around the element, on the outer edge of any padding. The border can be of several types, widths, and colors.
  • The padding exists around the content and inherits the background color of the content area.
  • The content is surrounded by padding.

One thing to keep in mind is that each margin,border, padding, and content will have independent width and height. Meaning, You will need to take into account of the width + padding-left + padding-right + border-left + border-right + margin-left + margin-right to get the true width size. The same principles apply with the height. You need to add padding-top + padding-bottom + border-top + border-bottom + margin-top + margin-bottom to get the true height.

There is another set of properties in the CSS box model: outline properties:

  • outline-width
  • outline-style
  • outline-color
  • outline-offset

The outline properties are similar to border properties but do not affect the box model or take up space. They are useful for adding visible edges to elements without changing the layout.

The box-sizing property allows us to change the padding, border, and margin. It takes in two values. The content-box and the border-box. Content-box is the default value, but we can use the border-box aka the quirks mode. Any defined padding and border are included inside the assigned width and height values.

We can start positioning the boxes by using the position property. The position property has 4 values it can hold. Relative, absolute, fixed, and static. From these values you can assign them to be placed left, right, top, or bottom except for static. You can also control they way elements are stacked to each other by utilizing the z-index.

Summary for Lesson 11

How the CSS Box Model Affects Lists

Although you can have list specific style types, you can change the overall list structuring by manipulating the margin, padding, color, and background-color. A cool trick to hide text but still show the properties of a link, is by offsetting the text-indent: -1000em; This will move the text way to the left of the web page.

Primary navigation takes users to the introductory pages of main sections of the site; secondary navigation takes users to pages within a certain section of the site. You can make them horizontal or vertial. When making list elements, it is a good idea to make a visual change to allow users to know it is a clickable element. To style lists that are nested in a list, specify which list you want to target. Example,nav ul and nav ul li would style the first level, but we can go deeper by using nav ul ul and nav ul ul li to target the second list/the one nested in.
