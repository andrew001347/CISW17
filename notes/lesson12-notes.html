Content presentation and behavior:

  • Content consists of the words that a visitor can read on your pages. You create the content as text and mark it up with HTML to define different classes of content—headings, paragraphs, links, and so on.
  • Presentation is the appearance and layout of the words, images, and other elements on each page—text formatting, fonts, colors, and graphics. This is where the power of CSS comes into play.
  • Behavior is what happens when you interact with a page—items that highlight when you move over them, forms you can submit, and so on. This is where JavaScript enters into the picture for enhanced front-end interactivity, along with server-side languages such as PHP and Ruby.

As the web continues to advance forward, we want to be mindful of graceful degradation. Some of the earliest web browsers did not support images in HTML. There are many different types of Third-Party JavaScript Libraries and Frameworks. jQuery is a library that allows for DOM manipulation and cross-browser coding features. The Prototype library simplifies tasks such as working with DOM objects, dealing with data in forms, and remote scripting (AJAX). This can be included with a simple Prototype.js file and with the <script src="prototype.js"></script> tag.

Using JavaScript Frameworks

JavaScript libraries are smaller, offering reusable code to enhance custom architectures, while frameworks are larger and enforce a specific structure, like the Model-View-Controller (MVC) pattern. In MVC, applications are divided into three components: Model (manages data and logic), View (displays information), and Controller (handles user interactions).

  • Model—Acts as the central component, even though it’s listed first in the name, holding application data, business rules, functions, and other logical elements.
  • View—Requests information from the model to show to the user.
  • Controller—Sends information to the model for processing through user interactions.
