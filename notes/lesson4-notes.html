Summary for Lesson 6

There are special characters you can use by typing out the code name. For exmaple the © is written with & followed by copy and ;. To boldface and italic text, use the strong and em tag respectively. The key difference of using these tags instead of the b and i tag, is that they not only change the apperaance of the text, but also the meaning to the text. The b and i tag are also less fexible when it comes to CSS and should be avoided. Additionally, a style can be implemented within the tag to add multiple different types of rules.

  • <sup></sup> — Superscript text, moves the text slightly up
  • <sub></sub> — Subscript text, slightly lower
  • <em></em> — Emphasized (typically italic) text
  • <strong></strong> — Strong (typically boldface) text
  • <pre></pre> — Monospaced text, preserving spaces and line breaks, keeps all the white space made.

There use to be a font tag, but with CSS style rules, we dont need a seperate tag change appearance. It can be written in a different file or with in the tag you want to manipulate. For example,<p style="font-family:arial, sans-serif, 'times roman'"> Some other formats we can use is font-size #; and color:blue ; The best way to do it however, it by linking it from your CSS file. From the style sheet we can use class and ID attributes to target and manipulate its appearance.

3 types of HTML lists

  • Ordered list: Start with <ol> and place the items with <li>
  • Unordered list" Start with <ul> and place the items with <li>
  • Definition list: Start with <dl> and place the items with <dt> and define them with <dd>

We can create tables by using the <table> tag. Once we have that we can create row with <tr>. Inside the tr, we can use the <th> tag to first create a heading that will automatically be centered. We can then start inputing data with <td>. So we first create the table with the tag, then the row, then insert the data. Use tr again to create a new row.

The article tag <article> is another tag where we can change the layout of the page to have columns. You can specify how many with the styles sheets.

Summary for Lesson 7

A way to reference other files/web content would be to use the href with the anchor tag <a href"link or directory"Displaying text> There is also a .. linking system where it simply means to move up one directory. It is best practice to stick to one case when naming your pages. Use lower case.

Linking within a page will include a # in the anchor tag. Then we can use the id attribute in the tag you want to reference to. Only when you want to move to a outside your webpage, you must include the https://. You can also link to different types of files such as .doc and .pdf. With these downloadable files, you can add the download attribute to make it downloadable.

An interesting anchor tool is to link it with your email. By using the href= mailto:____ you can make a premade email template.

EX: <a href="mailto:author@somedomain.com?subject=Book%20Question&cc=coauthor@somedomain.com&bcc=publisher@somedomain.com">Mail the author</a>

You can also control if you want to open the link in a new tab or stay on current tab and move to different content. By default it is by staying on the same tab, and moving to a different content. To open a new tab you simply put the target= attribute. EX: <a href="/some/file.html" target="_blank">Open a Window!</a>

Summary for Lesson 8

When picking colors to put in your web, be sure to use colors judiciously and with respect. This means being aware of culturally sensitive colors. Other things to consider is the use a natural palette of colors, using accessible color combinations, using colors with good contrast, especially for blocks of text, using a small color palette, and considering who your audience is.

There are two ways to pick your colors. You can use hexadecimals, or RGB/RGBa values. With hexadecimals, the color format is rrggbb. ff,cc,99,66,33,00 means full brightness, 80%, 60%, 40%, 33%, none respectively. A similar structure is followed with RGBs, but with numbers instead. 255, 204, 153, 102, 51, 0. Luckily, coding editors like VScode have a color picker, where we do not have to remember the color values.

When selecting images, be conscious of pixel dimensions.Most people accessing the web now have a computer with a screen that is at least 1024 × 768 pixels in resolution, if not considerably larger. Other popular resolutions are 1440 × 900 and 1366 × 768 pixels. Always be sure to include the alt description in the image tag. It is good HTML standards to adhear to.

  • <img> — Places an image file within the page.
  • Attributes/Styles:
    • style="background-color:color;" — Sets the background color of an element (such as <body>, <p>, <div>, <blockquote>, and other containers).
    • style="color:color;" — Sets the color of text within an element.
    • style="border:size type color;" — Sets the color of the four borders around an element. Border colors cannot be used without also specifying the width and type of the border.
    • src="address" — Gives the address or filename of the image.
    • alt="altdescription" — Provides an alternative description of the image, displayed when the image can't be viewed.
    • title="title" — Specifies a text message that is displayed as an image title, typically in a tooltip.
    • style="width:length;" — Specifies the width of the image (in pixels).
    • style="height:length;" — Specifies the height of the image (in pixels).
    • style="border:size type color;" — Removes the border around the image if it is a link.
    • style="vertical-align:alignment;" — Aligns the image vertically to text-top, top, text-bottom, bottom, middle, or baseline.
    • style="float:location;" — Floats the image to one side, allowing text to wrap around it. Possible values: left, right, none.
  • <map>…</map> — Defines a client-side image map referenced by <img usemap="...">. Includes one or more <area> tags.
  • <area> — Defines a clickable link within a client-side image map.
  • Attributes/Styles:
    • usemap="name" — Defines the name of an image map for client-side image mapping. Used with <map> and <area>.
    • shape="value" — Specifies the shape of the clickable area within the <area> tag (rect, poly, circle).
    • coords="values" — Specifies the coordinates of the clickable region within an image (varies by area type).
    • href="linkurl" — Specifies the URL that should be loaded when the area is clicked.
  • <audio>…</audio> — Plays an audio file natively in the browser.
  • Attributes/Styles:
    • src="mediaurl" — Provides the URL of the file to embed.
    • preload="preloadtype" — Tells whether to preload the media file (none, auto, metadata).
    • controls — Shows the audio player controls.
    • autoplay — Plays the file automatically when it finishes loading.
    • loop — Continuously plays the file until stopped.
  • <video>…</video> — Plays a video file natively in the browser.
  • Attributes/Styles:
    • src="mediaurl" — Provides the URL of the file to embed.
    • preload="preloadtype" — Tells whether to preload the media file (none, auto, metadata).
    • style="width:length;" — Specifies the width of the video (in pixels).
    • style="height:length;" — Specifies the height of the video (in pixels).
    • controls — Shows the video player controls.
    • autoplay — Plays the video automatically when loaded.
    • loop — Continuously plays the video until stopped.
