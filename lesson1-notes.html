Summary for Lesson 1

The Internet came to be when a group of scholars wanted to access and share information accross the states. Sir Tim Berners-Lee made a way to easily cross-reference text on the Internet through hypertext links. And thus his idea of HTML was made. Hypertext Markup Language is a language for describing how text, graphics, and files containing other information are organized and linked.

How we view web content is by having the text and images rendered by web browsers. 3 things are needed to view web content. HTML,CSS,and JavaScript.HTML describes the structure, CSS the design,and JavaScript defines the behavior of the content, or how it responds to different conditions.

To retrieve web content, a series of processes occur. In its simplest form, people get web content by requesting it to the web server by the web browser. The server would then send a response to the web browser that sends all the information the client/user requests. Browsers assemble the web content components and arrange those parts according to the HTML commands in the file.

In order to place HTML files on the web where it could be viewed online, a File Tranfer Protocol or,FTP, client is needed. The FTP client allows for your web content to be transfer to a web server.

An important thing to consider when developing web sites, is that not every person will view the contents of the web on the same screen, or even devices. It is important to make the website as responsive as possible to cover all cases of device usage.

Summary for Lesson 2

To create html files end the file's name with ".html". There are six elements that are fundamental in structuring a HTML document. Those six elements are <header>, <section>, <article>, <nav>, <aside>, and <footer>.

HTML Tags Every Webpage Must Have:

  • <!doctype> declaration.
  • <html></html> tag pair.
  • <head></head> tag pair
  • <title></title> tag pair
  • <body></body> tag pair
  • <meta charset="utf-8"><

Most HTML tags will have two parts. A opening tag < > and a closing tag. </ > Some may have an empty tag. An empty tag is a command without enclosing any text in the page.

You can organize a page with paragraphs(<p>) and line breaks(<br>). An interesting thing to note is that a line break tag does not require a closing tag. These tags can be used to separate the lines and make space between each tag. Additionally, a horizontal tag (<hr>) can also be utilized to create spacing.

Another method of organizing content would be the use of headings. Using different levels of headings shows a hierarchy of content. H1 tags defines the entire page. H2 tag defines sections withing the page and h3 tags can be used to define areas within the h2 tags. You can go up to h6 that can help with organizing content.

Using semantic elements provide additional meaning to not only people but also how machines interpret the document. This can be checked by utilizing THIS link.

21 HTML tags covered

  1. <html>…</html>: Encloses the entire HTML document.

  2. <head>…</head>: Encloses the head of the HTML document. Used within the <html> tag pair.

  3. <meta charset="utf-8">: Defines the character set for the document as UTF-8. Used within the <head> tag pair, immediately after the opening <head> tag.

  4. <title>…</title>: Indicates the title of the document. Used within the <head> tag pair and displayed at the top tab names and bookmarks.

  5. <body>…</body>: Encloses the body of the HTML document. Used within the <html> tag pair.

  6. <p>…</p>: Encloses a paragraph; skips a line between paragraphs.

  7. <br>: Single-spaced line break.

  8. <hr>: Draws a horizontal line.

  9. <h1>…</h1>: Encloses a first-level heading. Used to introduce the web.

  10. <h2>…</h2>: Encloses a second-level heading Displays infromation of lesser importance than heading 1.

  11. <h3>…</h3>: Encloses a third-level heading An additional sub heading.

  12. <h4>…</h4>: Encloses a fourth-level heading (seldom used).

  13. <h5>…</h5>: Encloses a fifth-level heading (seldom used).

  14. <h6>…</h6>: Encloses a sixth-level heading (seldom used).

  15. <header>…</header>: Contains introductory information that appear at the top of the page. However it can be used multiple times if your page content can be broken into sections.

  16. <footer>…</footer>: Contains additional material for its containing element (commonly a copyright notice or author information). Usually contains copyright and author information or links to related resources. Usually placed at the bottom.

  17. <nav>…</nav>: Contains navigational elements. These elemens will link to other sections within a site/page. Use these to link into key sections of the web page/content.

  18. <section>…</section>: Contains thematically similar content, such as a chapter of a book or a section of a page. This will not be independent content.

  19. <article>…</article>: Contains content that is a standalone body of work, such as a news articles, reviews, and other items that fit this description.

  20. <aside>…</aside>: Contains secondary information for its containing element Contains links to related resources, pull quotes, helper text, ect... .

  21. <address>…</address>: Contains address information related to its nearest <article> or <body> element, often contained within a <footer> element.
